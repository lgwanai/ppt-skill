# Agent-Per-Page PPT Generation

## Objective

Replace the monolithic hardcoded PPTX generator with an **Agent-per-page** architecture.
Each page is handled by an independent agent that orchestrates tools to:
1. Fetch page content from outline
2. Match to spec design
3. Plan visual assets and layout
4. Draw content
5. Review and iterate

All tool logic driven by LLM prompts — zero hardcoded rendering.

## Architecture

```
Observer (orchestrator)
  │
  ├── PageAgent (slide 0 - cover)
  ├── PageAgent (slide 1 - TOC)
  ├── PageAgent (slide 2 - content)
  ├── ...
  └── PageAgent (slide N - end)
         │
         ├── Tool: get_page
         ├── Tool: extract_spec
         ├── Tool: plan_assets
         ├── Tool: plan_layout
         ├── Tool: draw_zone
         ├── Tool: screenshot
         └── Tool: review
```

## Tools

### 1. get_page — Fetch Current Page Content
- **Input**: outline.md path, completion state
- **Output**: `{index, type, title, body_items, chapter}`
- **Logic**: Reads outline, finds first uncompleted page, marks as in-progress

### 2. extract_spec — Match Page to Spec Design
- **Input**: page content (type, title, body)
- **Output**: spec bundle `{palette, title_pos, title_font, title_size, body_pos, body_font, body_size, bg_color, bg_image, logo_pos, colors, shapes, decorations}`
- **Logic**: LLM analyzes page type (cover/toc/transition/content/end), searches spec directory for matching layout, returns ONLY style data (no content)

### 3. plan_assets — Plan Visual Elements
- **Input**: page content, spec, available assets (icons, diagram templates, images)
- **Output**: `{modules: [{type: "smartart"|"chart"|"diagram"|"image"|"text", contentType: "flow"|"compare"|"list"|"pyramid"|"cycle", contentRef: "which body items", estimatedCount: N}]}`
- **Priority**: SmartArt+text > SmartArt > chart+text > diagram+text > plain text
- **Limits**: max 2 graphics (SmartArt/chart/diagram) per page
- **Logic**: LLM analyzes content structure, decides optimal visual representation

### 4. plan_layout — Plan Page Layout Zones
- **Input**: page type, spec positions, asset modules
- **Output**: `{layout: "grid_2x2"|"left_right"|"top_bottom"|"center"|"six_grid", zones: [{x, y, w, h, content}]}`
- **Logic**: LLM calculates remaining space after fixed elements (title, nav), divides into zones based on content count and type

### 5. draw_zone — Draw Content into Zone
- **Input**: zone coordinates, content, spec styles, available assets
- **Output**: Updated PPTX slide
- **Logic**: Places text, icons, diagrams, images into the zone. Scales fonts and elements to fit. Uses text2image for dynamic images if needed. One call per zone.

### 6. screenshot — Screenshot Page
- **Input**: pptx path, page index
- **Output**: PNG path
- **Logic**: qlmanage or python-pptx render

### 7. review — Review Visual Quality
- **Input**: screenshot PNG, spec reference
- **Output**: `{pass: bool, issues: [{type: "overlap"|"spacing"|"alignment"|"overflow", detail, fix}]}`
- **Logic**: VL model analyzes screenshot for visual issues

## Execution Flow

```
for each page in outline:
    agent = PageAgent(page_index)
    content = agent.tool_get_page()
    spec = agent.tool_extract_spec(content)
    assets = agent.tool_plan_assets(content, spec)
    layout = agent.tool_plan_layout(content, spec, assets)
    
    for zone in layout.zones:
        agent.tool_draw_zone(zone, spec)
    
    for iteration in range(5):
        png = agent.tool_screenshot()
        review = agent.tool_review(png, spec)
        if review.pass: break
        agent.fix_issues(review.issues)
    
    agent.mark_complete()
```
