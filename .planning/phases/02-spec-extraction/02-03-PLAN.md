```python
class SpecExtractor:
    """Orchestrates PPTX spec extraction end-to-end."""

    def __init__(self, pptx_path: str | Path, spec_name: str):
        self.pptx_path = Path(pptx_path)
        self.spec_name = spec_name
        self.prs = Presentation(str(self.pptx_path))
        self.spec = DesignSpec()  # Start with defaults

    def extract(self) -> DesignSpec:
        """Run full extraction pipeline, return populated DesignSpec."""
        ...

    def save(self, specs_dir: str | Path = "specs") -> Path:
        """Extract and save YAML spec file. Returns file path."""
        ...
```

**`extract()` method implementation:**

1. **Metadata:** Set `spec.metadata = {"name": self.spec_name, "source": str(self.pptx_path.name), "extracted_at": datetime.now().isoformat(), "slide_count": len(self.prs.slides)}`

2. **Colors:** Load theme with python-pptx Presentation, then call `theme.extract_theme_colors(self.pptx_path)`. Map the 12 color keys (dk1, dk2, lt1, lt2, accent1-6, hlink, folHlink) into a `ColorPalette` dataclass instance. Any key not found in extraction → use config.py `DESIGN_COLORS` default value. Assign to `self.spec.colors`.

3. **Fonts:** Call `theme.extract_theme_fonts(self.pptx_path)`. Populate `Typography` dataclass: `heading_family = result["majorFont"]`, `body_family = result["minorFont"]`. After the per-slide loop completes, call `font_analysis.extract_all_slide_fonts(self.prs)` to get per-slide heading/body size data, then `font_analysis.compute_spec_typography_sizes(all_fonts)` to compute aggregate heading_sizes/body_sizes dicts (min/max/median per level). Assign to `self.spec.typography`.

4. **Slides (per-slide loop):** For each slide in `self.prs.slides`:
   - Classify using `slide_classifier.classify_slide(slide)`
   - Analyze layout using `layout_analysis.analyze_slide_layout(slide)`
   - Compute density using `density.analyze_slide_density(slide, i+1)`
   - Extract background using `theme.extract_slide_background(slide, self.pptx_path)`
   - Create a `SlideSpec` dataclass instance with all collected data
   - Append to `self.spec.slides`

5. **Density classification:** After computing all per-slide densities, call `density.classify_density(density_list)` to assign breathing/dense/anchor labels. Update each slide's density field.

6. **Font sizes:** Call `font_analysis.extract_all_slide_fonts(self.prs)` to get per-slide heading and body font size data (min/max/median from run→paragraph inheritance walk). Then call `font_analysis.compute_spec_typography_sizes(font_data)` to produce the final `Typography.heading_sizes` (title, subtitle, h1, h2 keys) and `Typography.body_sizes` (body, small, caption keys) dicts. Assign these to `self.spec.typography.heading_sizes` and `self.spec.typography.body_sizes` respectively. Handles empty text gracefully — empty dicts when no text found.

7. **Rhythm:** Call `density.build_presentation_rhythm()` passing density data and slide type sequence. Assign to `self.spec.rhythm`.

8. **Source config:** Set `self.spec.source_config` with canvas format hints. Derive from slide dimensions: if width/height ratio ≈ 1.778 → `{"canvas": "ppt169"}`, if ≈ 1.333 → `{"canvas": "ppt43"}`. Use EMU dimensions from `self.prs.slide_width` and `self.prs.slide_height`.

9. Return `self.spec`

**`save(specs_dir="specs")` method:**
- Call `self.extract()` if spec is empty
- Create `specs_dir` directory if not exists
- Write YAML using `yaml.dump(spec_data, f, default_flow_style=False, sort_keys=False, allow_unicode=True)`
- The `spec_data` needs dataclass→dict conversion. Use `_dataclass_to_dict()` helper that recursively converts dataclasses and enums.
- Return the Path of the written file

**Error handling:**
- Try/except around theme extraction (zipfile access). If theme XML not found → use config.py defaults.
- Try/except around slide iteration. Return partial spec if some slides fail — log warnings, don't crash.
- Missing theme1.xml → use DESIGN_COLORS defaults with a warning

**Serialization helper:** Since `yaml.dump` doesn't know about dataclasses, write a `_spec_to_dict(spec: DesignSpec) -> dict` function that uses `dataclasses.asdict()` (stdlib) for conversion, with a custom serializer for enums (extract `.value`). This ensures the YAML output has clean string values and no Python object references.

Research reference: RESEARCH.md §YAML Spec Serialization code examples. Use `dataclasses.asdict()` (stdlib) not manual dict construction.

Create CLI module at `src/ppt_skill/cli/` with `__init__.py` and `spec_commands.py`.

**`spec_commands.py`** should provide three standalone functions (not a class — follow the pattern from pipeline.py's function-based approach):

**`extract_spec(name: str, pptx_path: str, specs_dir: str = "specs") -> Path`**
- Instantiate `SpecExtractor(pptx_path, name)`
- Call `extractor.save(specs_dir)`
- Print to stdout: `✓ Spec "{name}" extracted from {pptx_path} → {output_path} (N slides)`
- Return the output Path

**`list_specs(specs_dir: str = "specs") -> list[str]`**
- Scan `specs_dir` for `*.yaml` files
- Read the YAML metadata from each to get slide count and extraction date
- Print a formatted table to stdout:
  ```
  Available specs:
    • corporate-blue     (12 slides, extracted 2026-01-15)
    • minimal-dark       (8 slides, extracted 2026-02-01)
  Active: corporate-blue
  ```
- Check `specs/.active` file for active spec name — mark it in output
- If no specs exist, print "No specs found in specs/ directory. Use extract-spec to create one."
- Return list of spec names (without .yaml extension)

**`select_spec(name: str, specs_dir: str = "specs") -> Path`**
- Verify `specs_dir/<name>.yaml` exists — if not, error out with "Spec '{name}' not found. Available: ..."
- Write the spec name to `specs_dir/.active` (plain text file, no YAML, just the name on one line)
- Print: `✓ Active spec set to: {name}`
- Return Path to `.active` file

**`get_active_spec(specs_dir: str = "specs") -> str | None`**
- Read `specs_dir/.active` if it exists
- Return spec name string or None
- Silently return None if `.active` doesn't exist (not an error)

All functions use `pathlib.Path` for paths. All output goes to stdout, errors go to stderr. No interactive stdin prompts — these are CLI commands.

The functions are designed to be called both from a CLI entry point (argparse) and programmatically from Phase 3-4 code.

**`__init__.py`:** Export the four functions via `__all__`.

**Integration note:** These functions are NOT hooked into the main CLI yet (that's Phase 5 packaging). For now, they are callable directly from Python. The `extract_spec` function is the primary user entry point for Phase 2 verification.
