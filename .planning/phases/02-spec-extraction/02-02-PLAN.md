Create `src/ppt_skill/spec/layout_analysis.py` with functions to measure spatial layout patterns from slide shapes.

**`analyze_slide_layout(slide) -> dict`**

Analyze a single slide's spatial layout and return:
```python
{
    "margins": {"top": float, "bottom": float, "left": float, "right": float},
    "title_position": {"x": float, "y": float, "width": float, "height": float} | None,
    "content_regions": [{"x": float, "y": float, "width": float, "height": float}],
}
```

**Margin calculation:**
- Get slide dimensions from `slide.slide_layout.slide_width` and `slide_height` (in EMU)
- For each non-background shape, record its bounding box (left, top, left+width, top+height)
- Top margin = minimum `top` across all shapes (in inches via `Emu(top).inches`, rounded to 2 decimal places)
- Bottom margin = `slide_height` - maximum `top + height` (in inches)
- Left margin = minimum `left` (in inches)
- Right margin = `slide_width` - maximum `left + width` (in inches)
- If no shapes exist, return slide dimensions as margin for all 4 sides

**Title detection:**
- Check `slide.shapes.title` â€” if present, record its `left`, `top`, `width`, `height` in inches (all rounded to 2 decimals)
- If no title shape but a text placeholder at the top of the slide exists, treat the highest text shape as title

**Content regions:**
- For every non-title, non-background shape, record its position and size in a list
- Group shapes by vertical position: if two shapes overlap vertically or are within 0.3 inches vertically, they share a content region

**`analyze_all_slides(prs) -> list[dict]`** â€” convenience function returning per-slide layout dicts with `slide_index` added.

Use `from pptx.util import Emu` for EMUâ†’inches conversion. All measurements in inches, rounded to 2 decimal places. Defensive: handle slides with zero shapes gracefully (return slide dimensions as margins).

Tolerate missing title â€” `title_position` can be None. Content regions can be empty list.

**`analyze_slide_density(slide, slide_index: int) -> dict`**

For a single slide, compute:
```python
{
    "slide_index": int,
    "char_count": int,      # sum of all text frame character lengths across shapes
    "image_count": int,     # count of shapes with image or Picture type
    "shape_count": int,     # total shapes on slide (len(list(slide.shapes)))
}
```

Character counting: iterate `slide.shapes`, for each shape with `has_text_frame`, sum `len(shape.text_frame.text)`. Handle shapes where `has_text_frame` is False or text_frame access raises â€” use try/except.

Image counting: check `hasattr(shape, 'image') and shape.image is not None`, OR check `shape.shape_type == MSO_SHAPE_TYPE.PICTURE (13)`.

**`classify_density(densities: list[dict]) -> list[dict]`**

Given per-slide density dicts from `analyze_slide_density`, classify each slide's density using **percentile-based thresholds** (not absolute character counts â€” this adapts to any deck length and writing style):

1. Extract all `char_count` values into a list
2. Compute the 20th and 80th percentiles of the distribution
3. For each slide:
   - `char_count <= 20th percentile` â†’ density = "breathing" (light content, visual rest)
   - `char_count >= 80th percentile` â†’ density = "anchor" (heavy, information-dense)
   - Between 20th and 80th â†’ density = "dense" (normal content level)
4. Edge case: if all slides have identical char_count (e.g., 0 across all), classify all as "breathing"
5. Return the original list with `"density": str` field added to each entry

**`programmatic_percentile(values: list[int], pct: float) -> float`** â€” pure Python percentile calculation (no numpy dependency). Use linear interpolation method (same as numpy's default).

**`build_presentation_rhythm(slide_densities: list[dict], slide_types: list[str]) -> PresentationRhythm`**
- From the ordered sequences of slide types and density labels, construct a `PresentationRhythm` dataclass:
  - `sequencing_pattern`: list of slide type strings in order
  - `density_profile`: list of density label strings in order
  - `story_arc`: count occurrences of each type: `{"opening": count_of_title + first_section, "development": count_of_content, "climax": count_of_data_or_dense, "closing": count_of_divider_or_last_slide}` â€” simple heuristic for v1

Import `DensityLabel`, `PresentationRhythm` from `spec_model`. Import `MSOSHAPE_TYPE` if needed via `from pptx.enum.shapes import MSO_SHAPE_TYPE`.
