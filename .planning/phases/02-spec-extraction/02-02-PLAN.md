**Strategy 1 — Layout name mapping (90% of slides):**
Define `LAYOUT_NAME_MAP` dict mapping standard PowerPoint layout names to types:
- 'Title Slide' → 'title'
- 'Title and Content' → 'content'
- 'Section Header' → 'section_divider'
- 'Two Content' → 'image_text'
- 'Picture with Caption' → 'image_text'
- 'Content with Caption' → 'image_text'
- 'Comparison' → 'content'
- 'Title Only' → 'content'
- 'Blank' → 'content'
- '1_Title Slide' → 'title' (common enterprise variant)
- 'Agenda' → 'content'
- 'Section' → 'section_divider'

For matching, use lowercase normalized comparison: `layout_name.strip().lower()` against keys normalized the same way. This handles case/space variations in custom templates.

If the normalized name matches a key, return the mapped type immediately.

**Strategy 2 — Content-based fallback (custom/unrecognized layouts):**
Use python-pptx shape inspection:
- Check for `has_chart` or `has_table` on any shape → return 'data'
- Check for images: `shape.shape_type == MSO_SHAPE_TYPE.PICTURE (13)` or hasattr(shape, 'image') → has_image = True
- Check for title: `slide.shapes.title is not None` → has_title = True
- Count placeholders: iterate shapes, check `shape.is_placeholder` and `shape.placeholder_format.type`
- If slide has images AND no title → 'image_text'
- If slide has a title but very few shapes (<= 3) → check if title text is short (< 30 chars) → 'section_divider' candidate
- If slide has title with more shapes → 'content'
- Default fallback → 'content'

**`classify_all_slides(prs) -> list[dict]`** — convenience function that iterates all slides, returns list of `{"slide_index": i+1, "type": type_str, "layout_name": raw_name}`

Import `SlideType` from `spec_model` but return string values (the enum's `.value`). Use `from pptx.enum.shapes import PP_PLACEHOLDER` for placeholder type checking.

  <name>Task 2: Create layout analyzer (layout_analysis.py) — spatial measurement from shape positions</name>
  <files>src/ppt_skill/spec/layout_analysis.py</files>
  <action>
Create `src/ppt_skill/spec/layout_analysis.py` with functions to measure spatial layout patterns from slide shapes.

**`analyze_slide_layout(slide) -> dict`**

Analyze a single slide's spatial layout and return:
```python
{
    "margins": {"top": float, "bottom": float, "left": float, "right": float},
    "title_position": {"x": float, "y": float, "width": float, "height": float} | None,
    "content_regions": [{"x": float, "y": float, "width": float, "height": float}],
}
```

**Margin calculation:**
- Get slide dimensions from `slide.slide_layout.slide_width` and `slide_height` (in EMU)
- For each non-background shape, record its bounding box (left, top, left+width, top+height)
- Top margin = minimum `top` across all shapes (in inches via `Emu(top).inches`, rounded to 2 decimal places)
- Bottom margin = `slide_height` - maximum `top + height` (in inches)
- Left margin = minimum `left` (in inches)
- Right margin = `slide_width` - maximum `left + width` (in inches)
- If no shapes exist, return slide dimensions as margin for all 4 sides

**Title detection:**
- Check `slide.shapes.title` — if present, record its `left`, `top`, `width`, `height` in inches (all rounded to 2 decimals)
- If no title shape but a text placeholder at the top of the slide exists, treat the highest text shape as title

**Content regions:**
- For every non-title, non-background shape, record its position and size in a list
- Group shapes by vertical position: if two shapes overlap vertically or are within 0.3 inches vertically, they share a content region

**`analyze_all_slides(prs) -> list[dict]`** — convenience function returning per-slide layout dicts with `slide_index` added.

Use `from pptx.util import Emu` for EMU→inches conversion. All measurements in inches, rounded to 2 decimal places. Defensive: handle slides with zero shapes gracefully (return slide dimensions as margins).

Tolerate missing title — `title_position` can be None. Content regions can be empty list.

  <name>Task 3: Create font analyzer (font_analysis.py) — walk run→paragraph→layout→master→theme for font sizes and weights</name>
  <files>src/ppt_skill/spec/font_analysis.py</files>
  <action>
Create `src/ppt_skill/spec/font_analysis.py` that extracts actual font sizes and weights per slide by walking the full style inheritance chain. This closes the SPC-01 gap — theme.py only extracts font families from theme1.xml; this module extracts the actual sizes/weights applied to text.

**`extract_slide_fonts(slide) -> dict`**

For a single slide, iterate all shapes with `has_text_frame`, then iterate `shape.text_frame.paragraphs`, then iterate each paragraph's `runs`. For each run:

1. **Font size resolution (inheritance walk):**
   - `size = run.font.size` — if not None, use it (returns `Pt` units, convert to float via `.pt`)
   - If None: check `paragraph.font.size` (the default paragraph font)
   - If still None: use a sentinel value of `None` and mark as "inherited" — the caller will report these separately
   - Collect all resolved sizes into a list

2. **Font weight detection:**
   - Check `run.font.bold` — if `True`, mark that font slot as bold
   - If `None` (inherited), check `paragraph.font.bold`
   - Record which font sizes are bold vs regular

3. **Heading vs body classification:**
   - If the shape is a title placeholder (`shape.is_placeholder and shape.placeholder_format.type == PP_PLACEHOLDER.TITLE (1)`) or the shape is `slide.shapes.title`, collect sizes as heading sizes
   - All other text shapes → body sizes
   - If a shape is at the top of the slide (top < 1.5 inches) and has large text (>28pt), treat as heading even if not a title placeholder

4. **Return format:**
```python
{
    "heading_sizes": {
        "min": float,     # smallest heading font size in pts
        "max": float,     # largest heading font size in pts
        "median": float,  # median heading font size in pts
    },
    "body_sizes": {
        "min": float,
        "max": float,
        "median": float,
    },
    "heading_bold": bool,  # True if any heading runs are bold
    "body_bold": bool,
    "all_heading_sizes": [float],  # raw list for percentile computation
    "all_body_sizes": [float],
}
```
   - Empty lists → return 0.0 for min/max/median fields (defensive)

**`extract_all_slide_fonts(prs) -> list[dict]`** — convenience function returning per-slide font dicts with `slide_index` added.

**`compute_spec_typography_sizes(all_fonts: list[dict]) -> dict`**

Given per-slide font extraction results, compute the aggregate Typography heading_sizes/body_sizes dicts:
- heading_sizes: `{"title": max_heading, "subtitle": median_heading, "h1": max_heading, "h2": median_heading}`
- body_sizes: `{"body": median_body, "small": min_body, "caption": min_body}`
- Use median for "typical" sizes and min/max for extremes
- These populate the `Typography.heading_sizes` and `Typography.body_sizes` fields

**`median(values: list[float]) -> float`** — pure Python median helper (no numpy dependency).

This module does NOT parse layout/master/theme XML for font defaults — it relies on python-pptx's `run.font.size` which already resolves inheritance from the PPTX object model. The only None values come from runs with truly no font size set anywhere. For those, use `ptx.util.Pt(18)` as the universal default body size.

Import `PP_PLACEHOLDER` from `pptx.enum.shapes`, `Pt` from `pptx.util`.

  <name>Task 4: Create density analyzer (density.py) — percentile-based breathing/dense/anchor classification</name>
  <files>src/ppt_skill/spec/density.py</files>
  <action>
Create `src/ppt_skill/spec/density.py` implementing content density and presentation rhythm analysis.

**`analyze_slide_density(slide, slide_index: int) -> dict`**

For a single slide, compute:
```python
{
    "slide_index": int,
    "char_count": int,      # sum of all text frame character lengths across shapes
    "image_count": int,     # count of shapes with image or Picture type
    "shape_count": int,     # total shapes on slide (len(list(slide.shapes)))
}
```

Character counting: iterate `slide.shapes`, for each shape with `has_text_frame`, sum `len(shape.text_frame.text)`. Handle shapes where `has_text_frame` is False or text_frame access raises — use try/except.

Image counting: check `hasattr(shape, 'image') and shape.image is not None`, OR check `shape.shape_type == MSO_SHAPE_TYPE.PICTURE (13)`.

**`classify_density(densities: list[dict]) -> list[dict]`**

Given per-slide density dicts from `analyze_slide_density`, classify each slide's density using **percentile-based thresholds** (not absolute character counts — this adapts to any deck length and writing style):

1. Extract all `char_count` values into a list
2. Compute the 20th and 80th percentiles of the distribution
3. For each slide:
   - `char_count <= 20th percentile` → density = "breathing" (light content, visual rest)
   - `char_count >= 80th percentile` → density = "anchor" (heavy, information-dense)
   - Between 20th and 80th → density = "dense" (normal content level)
4. Edge case: if all slides have identical char_count (e.g., 0 across all), classify all as "breathing"
5. Return the original list with `"density": str` field added to each entry

**`programmatic_percentile(values: list[int], pct: float) -> float`** — pure Python percentile calculation (no numpy dependency). Use linear interpolation method (same as numpy's default).

**`build_presentation_rhythm(slide_densities: list[dict], slide_types: list[str]) -> PresentationRhythm`**
- From the ordered sequences of slide types and density labels, construct a `PresentationRhythm` dataclass:
  - `sequencing_pattern`: list of slide type strings in order
  - `density_profile`: list of density label strings in order
  - `story_arc`: count occurrences of each type: `{"opening": count_of_title + first_section, "development": count_of_content, "climax": count_of_data_or_dense, "closing": count_of_divider_or_last_slide}` — simple heuristic for v1

Import `DensityLabel`, `PresentationRhythm` from `spec_model`. Import `MSOSHAPE_TYPE` if needed via `from pptx.enum.shapes import MSO_SHAPE_TYPE`.
