<task type="auto">
  <name>Task 2: Implement theme extraction (theme.py) — colors, fonts, backgrounds via lxml</name>
  <files>src/ppt_skill/spec/theme.py</files>
  <action>
Create `src/ppt_skill/spec/theme.py` implementing theme-level extraction functions. Use `lxml.etree` and `zipfile` — do NOT rely on python-pptx's high-level theme API (returns None for inherited values).

**Function: extract_theme_colors(pptx_path: str) -> dict[str, str]**
- Open PPTX as zipfile, read `ppt/theme/theme1.xml`
- Parse with `lxml.etree.fromstring()`
- Navigate to `a:themeElements/a:clrScheme` (namespace: `http://schemas.openxmlformats.org/drawingml/2006/main`)
- For each child of clrScheme: extract localname as key (dk1, dk2, lt1, lt2, accent1-6, hlink, folHlink)
- For the inner color element: if `srgbClr`, use `val` attribute prefixed with `#`; if `sysClr`, use `lastClr` or fallback to `val`; if `schemeClr`, resolve against clrScheme's own entries
- Return dict mapping scheme name → HEX string (e.g., `{"dk1": "#000000", "accent1": "#4472C4"}`)

**Function: extract_theme_fonts(pptx_path: str) -> dict[str, str]**
- Read `ppt/theme/theme1.xml`
- Navigate to `a:themeElements/a:fontScheme`
- Extract `majorFont/a:latin/@typeface` and `minorFont/a:latin/@typeface`
- Return `{"majorFont": "Calibri", "minorFont": "Calibri"}`

**Function: extract_slide_background(slide, pptx_path: str) -> dict | None**
- **CRITICAL: Avoid python-pptx bug #1126.** Do NOT iterate children of `slide.background._element` — it returns `<p:cSld>`, not `<p:bg>`.
- Use: `cSld = slide.background._element` then `bg_elem = cSld.find(f'{{{P_NS}}}bg')` where P_NS = `http://schemas.openxmlformats.org/presentationml/2006/main`
- If `p:bg` element exists, check `p:bgPr/a:solidFill` for srgbClr (return `{'type': 'solid', 'color': '#XXXXXX'}`)
- If `p:bgRef` exists with schemeClr, return `{'type': 'theme_ref', 'ref': 'dk1'}`
- If no `p:bg`, check slide layout XML: open pptx zip, read the slide layout's XML (path from slide.slide_layout), find `p:bg` there
- If still no bg, check slide master XML (from layout's type attribute, path in `ppt/slideMasters/`)
- If still no bg, check theme's `bgFillStyleLst` (fallback)
- Return None only if absolutely nothing found AND mark `inherited_from: "theme"` in such cases
- Also detect gradients: check for `a:gradFill` with `a:gsLst` stops — if found, extract stop positions and colors, flag as `{'type': 'gradient', 'stops': [...], 'path_type': 'linear'|'circle'}`

Use the RESEARCH.md code examples (§Theme Color Extraction, §Theme Font Extraction, §Background Extraction) as templates. Add defensive None-checking throughout — malformed XML should not crash the extractor.

Verify by testing against the provided thematic-approach.pptx fixture (if available) or any real .pptx with a theme.
