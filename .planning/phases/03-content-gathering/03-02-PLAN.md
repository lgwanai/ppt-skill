def assess_sufficiency(

**Implementation approach:**

This function implements a deterministic heuristic for evaluating whether user input is detailed enough to skip questioning. It does NOT call an LLM directly — it implements the scoring rubric that Claude (the AI runtime) follows. The actual LLM-based assessment is driven by a prompt template defined as a module-level constant that Claude reads and executes.

**The scoring rubric (module-level constant for reference):**

```python
SUFFICIENCY_RUBRIC = """
Evaluate the following content input for a presentation. Score each dimension 0-2:

1. STRUCTURE (0-2): Are topics/sections clearly delineated?
   0 = no structure visible
   1 = some grouping implied (related topics together)
   2 = numbered/headered sections, explicit outline formatting, or slide-like structure

2. DETAIL (0-2): Does each section have supporting content?
   0 = topics only, no elaboration
   1 = some bullets or brief descriptions present
   2 = rich content with specific examples, data, or detailed explanations

3. AUDIENCE (0-2): Is target audience and purpose clear?
   0 = not stated or implied
   1 = vaguely implied by content type or domain
   2 = explicitly stated (e.g., "for executives", "training new hires")

4. SCOPE (0-2): Is the presentation scope bounded?
   0 = open-ended, unbounded topic
   1 = rough scope implied (time period, department, project)
   2 = well-defined, bounded scope

Threshold for SUFFICIENT:
- Total score >= 5
- STRUCTURE >= 1 (must have at least some organization)
- DETAIL >= 1 (must have at least some content depth)

Input:
{user_input}
"""
```

**The `assess_sufficiency()` function:**

1. Takes `user_input: str` and optional `spec_context: dict | None` (if a spec is active, its slide type distribution may inform structure expectations)
2. Returns a `SufficiencyResult` dataclass

**Function body:**

Since this is a prompt-engineering module (Claude processes the prompt, not Python), the Python function provides:

a) **Prompt template access:** Returns the rubric with the user's input embedded for Claude's evaluation. Store the rubric as a module constant and provide it via `get_sufficiency_prompt(user_input)`:

```python
def get_sufficiency_prompt(user_input: str) -> str:
    """Return the formatted sufficiency assessment prompt with embedded input."""
    return SUFFICIENCY_RUBRIC.replace("{user_input}", user_input)
```

b) **Result construction helper:** `build_sufficiency_result(scores: dict[str, int], rationale: str, section_count: int, estimated_slide_count: int) -> SufficiencyResult`:
   - Computes `total = sum(scores.values())`
   - Determines `sufficient = total >= 5 and scores.get("structure", 0) >= 1 and scores.get("detail", 0) >= 1`
   - Computes `confidence = min(total / 8.0, 1.0)` (max possible = 8)
   - `missing_dimensions = [dim for dim, score in scores.items() if score < 2]`
   - Returns populated `SufficiencyResult`

c) **The main `assess_sufficiency()` function:**
   - Calls `get_sufficiency_prompt(user_input)` to get the prompt
   - Claude (the AI runtime) evaluates the prompt and returns structured scores
   - The function wraps the result in `SufficiencyResult` using `build_sufficiency_result()`
   - If `spec_context` is provided, include it in the prompt for additional context (e.g., "Active spec uses 40% content slides — expect similar structure")

**Document the prompt template clearly** — future tuning adjusts the rubric wording, not the code. Place the rubric constant at module top with clear comments.

**Edge cases:**
- Empty input → all scores 0, sufficient=False, missing_dimensions all 4
- Single-word topic ("Sales") → structure=0, detail=0, audience=0, scope=0, sufficient=False
- Full outline pasted → structure=2, detail=2, audience=1+, scope=1+, typically sufficient=True

Export `assess_sufficiency`, `get_sufficiency_prompt`, `build_sufficiency_result`, and `SUFFICIENCY_RUBRIC` in `__all__`.

**Module-level constants:**

```python
# Priority order for question categories
QUESTION_CATEGORY_PRIORITY = [
    "structure",     # What are the main sections/topics?
    "detail",        # What content goes in each section?
    "audience",      # Who is this for, what's the takeaway?
    "storytelling",  # What's the narrative arc?
]

# Templates for section-level overview questions (Phase 1: one per section)
SECTION_QUESTION_TEMPLATES = {
    "structure": "What key topics or points should the '{section}' section cover?",
    "detail": "What specific content, examples, or data should appear in '{section}'?",
    "audience": "What should the audience understand or do after the '{section}' section?",
    "storytelling": "How should '{section}' transition from the previous section?",
}

# Templates for gap-fill questions (Phase 2: targeted follow-ups)
GAP_QUESTION_TEMPLATES = {
    "missing_detail": "Can you provide 2-3 specific points or examples for '{section}'?",
    "no_examples": "What concrete example or case study illustrates '{section}'?",
    "no_takeaway": "What's the single key takeaway for the '{section}' section?",
    "missing_transition": "How does '{section}' connect to the next topic?",
}
```

**Public functions:**

**`generate_section_questions(session: QuestionSession, sections: list[str], context: str = "") -> list[Question]`**

Phase 1: Ask one overview question per identified section, consuming from the budget.

- For each section in `sections`:
  - If `not session.can_ask()`: stop (budget exhausted)
  - Determine category: first section uses "structure", subsequent sections use "detail"
  - Create a `Question` with: `id = len(session.questions_asked) + 1`, `category`, `text` from the template, `target_section = section`, `context` describing why this question is asked
  - Call `session.mark_asked(question)`
- Return the list of generated questions

**`generate_gap_questions(session: QuestionSession, gaps: dict[str, list[str]], context: str = "") -> list[Question]`**

Phase 2: Allocate remaining budget to sections with most gaps first.

- If `not session.can_ask()`: return empty list
- Sort sections by gap count (descending) — sections with more gaps get questions first
- For each section in sorted order:
  - If `not session.can_ask()`: stop
  - Take the first gap type from `gaps[section]`; look up template in `GAP_QUESTION_TEMPLATES`; fallback to "missing_detail" template if gap type not found
  - Create `Question` with appropriate id, category="detail" (gap-fill questions are always detail category), `text`, `target_section=section`, `context` explaining the gap
  - Call `session.mark_asked(question)`
  - Remove the addressed gap from the list so subsequent rounds target different gaps
- Return the list of generated questions

**`identify_content_gaps(known_content: dict[str, list[str]], sections: list[str]) -> dict[str, list[str]]`**

Helper function that analyzes what's known about each section and identifies gaps.

- `known_content: dict[str, list[str]]` — keys are section names, values are lists of known facts/points
- For each section:
  - If no content known: gaps = ["missing_detail", "no_examples", "no_takeaway"]
  - If only 1-2 points: gaps = ["no_examples", "no_takeaway"]
  - If no examples/data: gaps = ["no_examples"]
  - If no takeaway: gaps = ["no_takeaway"]
  - For last section + first section: add "missing_transition" if applicable
- Return `gaps: dict[str, list[str]]` with section name → gap type list mapping

**`get_total_questions_asked(session: QuestionSession) -> int`**

Simple accessor: `return len(session.questions_asked)`.

**Design notes:**
- Questions are NEVER generated beyond the budget — `can_ask()` gate on every question
- The `session` object is MUTATED by these functions (budget decremented, questions appended)
- Callers should create a fresh `QuestionSession()` per gathering session
- Question text uses `str.format(section=...)` — section names with special characters should be sanitized by the caller
- Export all public functions in `__all__`
