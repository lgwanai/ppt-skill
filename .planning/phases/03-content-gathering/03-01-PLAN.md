class SufficiencyResult:

    missing_dimensions: list[str] = field(default_factory=list)
    section_count: int = 0
    estimated_slide_count: int = 0
    scores: dict[str, int] = field(default_factory=dict)
    rationale: str = ""

**Part C — `QuestionSession` dataclass:**

```python
@dataclass
class QuestionSession:
    """Tracks state across an adaptive questioning session.
    
    Budget starts at 8 questions. Section-level questions are asked first
    (one per section), then remaining budget is allocated to gap-fill
    questions for sections with identified knowledge gaps.
    """
    questions_asked: list[Question] = field(default_factory=list)
    budget_remaining: int = 8
    sections_identified: list[str] = field(default_factory=list)
    gaps_per_section: dict[str, list[str]] = field(default_factory=dict)

    def can_ask(self) -> bool:
        """True if budget remains for more questions."""
        return self.budget_remaining > 0

    def mark_asked(self, question: Question) -> None:
        """Record a question as asked, decrement budget."""
        self.questions_asked.append(question)
        self.budget_remaining -= 1
```

Add a helper property or method `QuestionSession.total_asked` that returns `len(self.questions_asked)`.

Update `__all__` in `__init__.py` to include the new types (already listed above, just verify).

Update the module docstring to describe all types.

**Design decisions (from RESEARCH.md):**
- `Question.category` uses the priority ordering: structure > detail > audience > storytelling
- `QuestionSession.gaps_per_section` keys are section names, values are lists of gap types ("missing_detail", "no_examples", "no_takeaway", "missing_transition")
- `QuestionSession.budget_remaining` starts at 8 and decrements with `mark_asked()`
- No validation in the dataclass itself — validation happens in questioning.py when generating questions
