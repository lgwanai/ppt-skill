class ContentGatherer:
    """Orchestrates content gathering: sufficiency assessment → adaptive questioning → outline generation.

        """Run full content gathering pipeline. Returns ContentOutline."""
        ...

    def save(self, outlines_dir: str | Path = "outlines") -> Path:
        """Save the generated outline as YAML. Returns file path."""
        ...

**`gather()` method — 3-phase pipeline:**

**Phase 1 — Sufficiency Assessment:**
1. Load spec context if `spec_name` is provided (or auto-resolve via `get_active_spec()` from `ppt_skill.cli.spec_commands`)
2. Call `assess_sufficiency(user_input, spec_context)` to get `SufficiencyResult`
3. Store result in `self.sufficiency`
4. If `mode == "skip_questions"` OR `result.sufficient`: proceed directly to Phase 3
5. Otherwise: proceed to Phase 2

**Phase 2 — Adaptive Questioning:**
1. Create `QuestionSession()` (fresh 8-question budget)
2. Extract sections from user_input (Claude identifies topic sections from the input text) — store in `session.sections_identified`
3. Call `generate_section_questions(session, sections, context)` — one overview question per section
4. For each section question: Claude presents the question to the user and collects the answer. Store answers in `self.gathered_content[section]` as a list of facts/points.
5. Call `identify_content_gaps(self.gathered_content, sections)` to find remaining gaps
6. Call `generate_gap_questions(session, gaps, context)` to allocate remaining budget
7. For each gap-fill question: present to user, collect answer, update `self.gathered_content`
8. After all questions answered (or budget exhausted): proceed to Phase 3

**Phase 3 — Outline Generation:**
1. Compile a content summary from `self.gathered_content` plus any additional context from the user's original input
2. Generate a `ContentOutline` with `presentation_title`, sections, and per-slide `SlideEntry` instances
3. Apply layout type rules:
   - First slide → `OutlineLayoutType.TITLE`
   - Section-start slides (first slide of each section) → `OutlineLayoutType.SECTION_DIVIDER`
   - Content slides (bullet-heavy) → `OutlineLayoutType.CONTENT`
   - Comparison/contrast content → `OutlineLayoutType.TWO_COLUMN`
   - Slides with image_hints → `OutlineLayoutType.IMAGE_TEXT`
4. Validate via `outline.validate()` — if invalid, regenerate with the issues as feedback
5. Store in `self.outline` and return

**Implementation note — LLM interaction:**

The ContentGatherer itself is a state machine and prompt orchestrator. Claude (the AI runtime) processes the prompts. The Python code provides:

- Prompt templates for each phase (module-level constants)
- State tracking (budget, gathered content, session)
- Validation and quality gates
- YAML serialization

Define these prompt templates as module-level constants:

```python
OUTLINE_GENERATION_PROMPT = """

Context:
- Active spec: {spec_name}
- Available layout types: title, content, two_column, section_divider, image_text, data
- Target audience: {audience}
- Purpose: {purpose}

Content gathered:
{content_summary}

Rules:
1. First slide MUST be a TITLE slide with presentation title and subtitle
2. Each section starts with a SECTION_DIVIDER slide (section name only)
3. Content slides use CONTENT layout type by default
4. Use TWO_COLUMN for comparison/contrast content (pros/cons, before/after, etc.)
5. Use IMAGE_TEXT for slides that would benefit from imagery
6. Every slide MUST have a descriptive title AND at least 2 body points with >10 characters each
7. Total slides: follow the gathered content scope (don't pad or truncate significantly)

Output format: JSON array of slide objects:
[
  {{"title": "...", "body": ["...", "..."], "layout_type": "...", "notes": "...", "image_hint": "...", "section_name": "..."}},
  ...
]
"""
```

**`save()` method:**
- If `self.outline` is None, raise ValueError
- Create `outlines_dir` directory if not exists
- Set outline metadata: `name` (derived from title, kebab-case), `saved_at` (ISO datetime), `spec_name`, `question_count`
- Serialize via `_dataclass_to_dict(self.outline)` → `yaml.dump()` with same settings as Phase 2: `default_flow_style=False, sort_keys=False, allow_unicode=True`
- Write to `outlines/<name>.yaml`
- Return the Path

**Static helper: `load_outline(name, outlines_dir="outlines")` → ContentOutline**
- Read `outlines/<name>.yaml`, parse with `yaml.safe_load()`, reconstruct via `ContentOutline.from_dict()`

**Edge cases:**
- Empty user input → raise ValueError with helpful message
- Zero sections identified → generate a minimal outline with title slide only + warning flag
- All 8 questions exhausted but gaps remain → generate outline with best available content, add `quality_warning` to metadata
- `mode="skip_questions"` but input is truly insufficient → generate best-effort outline, flag in metadata

**Document all prompt templates clearly** — they are the primary tuning surface for output quality.
