## 三、让素材会说话：用豆包写图片提示词（8分钟）

**为什么讲这个？** PPT 不是只有文字，背景图、插画、icon 决定了视觉质量，而产品经理往往卡在“不知道怎么描述想要的图”。

### 3.1 写提示词的“四要素”框架

```
主体（谁/什么） + 场景/环境 + 风格/质感 + 用途约束
```

**案例对比**：

❌ **弱提示词**：“一张商务背景图”
✅ **强提示词**：“现代简约风格的蓝色抽象几何背景，带有柔和的渐变和微妙的网格线条，适合金融科技 PPT 封面使用，16:9 比例，无文字，留白区域在右侧 40%”

### 3.2 三种 PPT 常用素材的提示词模板

**① 封面背景**
> “宽屏商务科技感背景，深蓝到紫色渐变，细线条组成网络节点图，低对比度，画面右侧留白用于放置白色标题文字，16:9，4K 画质，干净无文字”

**② 插画/插图**
> “扁平化矢量插画风格，一位职场人士在电脑前分析数据图表，简洁色块、圆角、暖色调点缀，透明背景，适合放在 PPT 右侧内容区，无文字”

**③ Icon**
> “线性图标风格，钱包/手机/盾牌的组合图形，2px 描边，#4472C4 色，白色背景正方形画布，适合做 PPT 列表项的项目符号，简洁专业”

**要点**：告诉学员——“用途约束”这一句是最关键的，它决定了 AI 生成的图能不能直接贴进 PPT。

## 四、一个刺痛所有人的问题（5分钟）

**讲师现场发问**：
> 既然豆包、Kimi 这些工具一句话就能出 PPT，那我们为什么还要费劲学这么多？直接生成不就完了吗？

**展示对比**：

- 左边：工具生成的 PPT（排版随机、配色花哨、字体不可控、没有公司 Logo，内容逻辑浅）
- 右边：公司内部正式汇报的 PPT（企微模板、统一字体配色、封面/目录/正文/尾页各有固定布局，内容有严格的叙事逻辑）

**引爆问题**：
> 工具生成的 PPT **质量达不到直接使用的程度**，根本原因是什么？

**三个根本矛盾**：
1. **视觉规范冲突**：每家公司的 PPT 模板都有特定的调色板、字体组合、Logo 位置、装饰元素——AI 生成完全不知道这些。
2. **布局逻辑缺失**：封面、目录、过渡页、正文、尾页，各自的元素角色不同（标题、副标题、正文、图表、页码），AI 只是“把文字塞进去”，不理解“这个位置就应该是标题，那个位置是注释”。
3. **叙事结构不匹配**：企业汇报讲究“论点→论据→总结”的推进逻辑，AI 生成的是信息罗列。

**自然引出**：
> 那能不能让 AI **先学习我们公司的模板**，再根据我们的大纲，**自动生成符合规范的 PPT** 呢？答案是：**可以，而且我们已经做出来了。**

## 五、破局方案：ppt-skill 的核心思路（5分钟）

**一句话定义**：
> ppt-skill 是一个 **“先理解设计，再生成内容”** 的 AI 技能，它从你已有的 PPT 模板中**提取完整的布局蓝图**，然后用 **Agent 自迭代机制**生成高质量的、可直接编辑的 .pptx 文件。

**核心三步**（用浅显语言讲）：

**Step 1：提取设计“基因”**
- 不是简单记住“标题字号大一点”，而是用 **VL 模型（视觉语言模型）** 同时看 PPT 截图 + 底层元素数据，判断每个元素的“角色”：谁是标题、谁是正文、谁是装饰、谁和谁组成了一个卡片。
- 就像一位设计师拿着你的模板说：“我明白了，你们的封面左对齐大标题，右上角有 Logo，正文页有两种布局：图片左文字右，和纯文字三栏。”

**Step 2：准备内容大纲**
- 你用简单的 YAML 或文字把你想要的内容列出来，不需要关心设计。

**Step 3：Agent 循环生成**
- AI 不是一次性生成完事，而是 **生成 → 自检 → 修正 → 再生成**，直到质量达标。这就是 Agent 的核心，接下来我们重点展开。

## 六、核心重点：Agent 机制深度解析（15分钟）

> 这是今天最重要的部分。前面讲工具、讲技巧，都是“术”；Agent 是“道”，是 AI 应用正在发生的底层范式变化。

### 6.1 从“一句话生成”到“Agent Loop”

**传统 AI 生成流程**：
```
用户输入 → AI 处理 → 直接输出结果
```
- 像快递：你把要求给出去，回来什么就是什么，没有检查、没有修改。

**Agent Loop 流程**：
```
用户输入 → AI 生成初稿 → 评估系统打分 → 得分 < 90%？
                                      ↓ 是
                              提取问题 → 生成修复指令 → AI 重生成
                                      ↓ 否
                                    输出最终结果
```
- 像匠人：先做一个胚子 → 拿卡尺量 → 不对就修 → 再量 → 修到符合标准为止。

**在 ppt-skill 中的具体实现**：

每一页 PPT 的生成都是一个 **“生成→评估→修复”** 的循环：

1. **生成**：将大纲内容 + 模板风格指南打包为 prompt，调用大模型生成 SVG（页面描述）。
2. **评估**：一个叫 `StyleEvaluator` 的“质检员”从 **五个维度** 打分：
   - 🎨 **颜色（30%）**：用到的颜色是否在模板规定的调色板内？
   - ✍️ **排版（20%）**：字体种类、大小层级和模板是否一致？
   - 📐 **布局（30%）**：标题位置、正文区域、图片位置的 IoU（交并比）是否对齐？
   - 🖼️ **背景（15%）**：背景色和背景图是否正确？
   - 📊 **密度（5%）**：内容疏密程度是否匹配（呼吸感 vs 信息密集型）？
3. **修复**：如果综合得分低于 90%，评估系统会把发现的问题变成“修改指令”反馈给生成模型，比如说：“标题字体应该用微软雅黑 24pt，现在用了宋体 18pt，请修正；右侧图片区域留白不足，请将正文宽度缩减至 55%。”
4. **迭代**：这个过程最多进行 **5 轮**，最终选取得分最高的版本。

**为什么是 5 轮而不是 1 轮？** 现场展示一个实际案例的数据：
- Round 1：得分 72%（颜色错误多，布局偏了）
- Round 2：得分 85%（颜色修好了，字体还有问题）
- Round 3：得分 93%（通过，输出）

> 没有 Agent Loop，用户拿到的是 72 分的“废品”；有了 Agent Loop，自动迭代到 93 分的“可交付品”。

### 6.2 什么是 Agent？——非技术视角的定义

**给出一个简洁的、产品经理能懂的定义**：
> Agent（智能体）= **有目标的 AI + 能使用工具 + 会自我反思的循环系统**

四个核心能力：
- **感知**：能“看”截图，能“读”元素属性（VL 模型分析）。
- **规划**：根据内容大纲，选择合适的布局模板，匹配元素位置。
- **行动**：调用工具生成 SVG，转换为原生 PowerPoint 形状。
- **记忆/反馈**：通过评估得分记住问题，在下一轮修正。

**类比**：传统编程像流水线工人——按照固定工序一步步做，做完就交。Agent 像一个学徒工匠——边做边检查自己的活，发现不对就调整，直到师傅（质量标准）满意。

### 6.3 Human-in-the-Loop：人什么时候介入？

Agent 自动迭代已经很强大，但在关键决策点上，**人的判断仍然不可替代**：

| 阶段 | Agent 做什么 | 人做什么 |
|------|-------------|---------|
| 规范提取 | 自动分析模板元素角色 | 确认封面/正文/尾页的分类是否正确 |
| 内容准备 | 生成内容大纲建议 | 审核、修改叙事逻辑，补充业务观点 |
| 生成迭代 | 自动循环修复到 90%+ | 查看最终版本，做最后的文案微调 |
| 特殊页面 | 自动排版 | 对核心数据页、客户案例页手动介入调整 |

**Human-in-the-Loop 的原则**：
> Agent 负责“做到 90 分”，人负责“从 90 分到 100 分的画龙点睛”。

**这在企业场景特别重要**：产品经理的行业知识、对领导偏好的理解、对微妙表述的把握，是 AI 无法替代的。Agent 帮你省掉 80% 的重复劳动，剩下 20% 的决策需要你来把关。

### 6.4 与传统流程式研发的核心区别

| 维度 | 传统流程式研发 | Agent 模式 |
|------|---------------|-----------|
| **核心逻辑** | 定义规则 → 执行 → 输出（线性的） | 定义目标 → 生成 → 评估 → 修正（循环的） |
| **异常处理** | 遵循预设的 if-else，遇到未定义情况就失败 | 自我感知问题，自主规划修正方案 |
| **质量保证** | 依赖测试阶段发现问题，反馈周期长 | 每次生成即质检，实时修复 |
| **迭代成本** | 改需求 → 改代码 → 重新测试，周期以天计 | 改大纲 → 重新生成，周期以分钟计 |
| **可扩展性** | 新增一种 PPT 布局需要程序员写代码 | 提供一个新模板 PPT，Agent 自动学会 |

**一句话总结**：
> 传统研发是 **“人把所有规则写死”**，Agent 是 **“人定义目标和质量标准，AI 自己找路径、自己修正”**。

## 七、Agent 的应用场景与未来展望（5分钟）

### 7.1 除了做 PPT，Agent 还能干什么？

**贴近产品经理日常的场景举例**：

- **PRD 生成与合规检查**：Agent 读取需求一句话，生成完整 PRD，再对照公司模板和合规 checklist 自动打分修改，直到符合发布标准。
- **数据分析报告**：Agent 连接数据库，自动写 SQL、生成图表、撰写分析结论，发现问题数据时自动钻取验证。
- **竞品监控**：Agent 定时抓取竞品更新日志、舆情，自动生成竞品简报，对于负面信息自动标记预警。
- **自动化测试**：Agent 读取产品页面截图，对比设计稿，发现 UI 偏差，自动生成 Bug 报告。

### 7.2 Agent 改变了什么？

**研发范式的转移**：

- **从“流程自动化”到“认知自动化”**：以前我们自动化的是“重复性动作”（比如批量重命名文件），Agent 自动化的是“判断和修正”（比如判断一个按钮的颜色对不对）。
- **从“软件服务人”到“人指导软件”**：人从操作者变成监督者和决策者。
- **反馈循环成为核心设计模式**：Loop（循环）不再是一个 Bug（比如死循环），而是 Agent 架构的**第一性原理**。

**给产品经理的启示**：
> 未来你设计产品时，可能不再只是画出页面跳转流程图，而是在设计“给 AI Agent 的指令和评估标准”。掌握 Agent 思维，是产品经理 AI 转型的关键一步。

## 八、总结与 Q&A（5分钟）

**今日核心回顾**：

1. 市面上有很多 PPT 生成工具，便捷但不满足企业规范要求。
2. 写好图片提示词，可以让 AI 为你的 PPT 生成合用的素材。
3. **真正的突破**在于让 AI “先学习你的设计规范，再生成内容”——ppt-skill 解决了这个问题。
4. **Agent Loop（生成→评估→修正）** 是这个解决方案的灵魂，它让 AI 从一个“一次性输出机器”变成了“会自我修正的工匠”。
5. Agent 正在重塑软件研发，从线性流程变为循环自优化，而人在循环中扮演把关和升华的角色。
6. **Human-in-the-Loop** 的理念将贯穿未来的 AI 产品设计。

**结束语**：
> 四节课下来，我们从“怎么和 AI 说话”走到了“如何让 AI 像一个负责任的团队成员一样工作”。记住，AI 不是替代你的判断力，而是把你的判断力放大。接下来，就轮到你们在公司里做出真正能用的 AI 产品了。
